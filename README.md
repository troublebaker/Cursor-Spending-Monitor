## 📖 How to Use

### Step 1 — Click **Update** for daily usage

![Update button](docs/screenshots/btn-update.png)

Click **Update** to quickly scrape your Cursor dashboard. Within seconds you'll see:
- How many API calls you made each day
- Which days were heavy vs light usage
- Your plan quota consumption progress

### Step 2 — Click **Update+Token** for full token breakdown

![Update+Token button](docs/screenshots/btn-update-token.png)

Click **Update+Token** to additionally capture the hidden token details on each row.  
This takes **3–4 minutes** (data updates in real time as it processes each row), but gives you:
- Exact **CacheR / CacheW / Input / Output** token counts per request
- Approximate **cost per call** based on token usage
- The full picture of *why* your bill is what it is

> ⚠️ Update+Token hovers each row to reveal hidden data — this is why it takes longer. Progress updates in real time so you can watch it work.

## 📊 What You'll See

### Plan & Quota — Know your subscription at a glance

![Plan & Quota](docs/screenshots/plan-quota.png)

Shows your current plan (e.g. Ultra $200/mo), reset date, and how much of each quota you've used:
- **Included 总用量** — overall included-quota consumption (e.g. 13%)
- **Auto + Composer** — usage by Auto/Composer models
- **API** — usage by direct API calls (e.g. 31%)
- **ON-DEMAND & LIMIT** — on-demand spending vs your cap ($0.00 / $200)

---

### Daily Calls — See which days you worked hardest

![Daily Calls chart](docs/screenshots/daily-calls.png)

Stacked bar chart of daily API call count, split by:
- **Included** (covered by your plan quota)
- **On-Demand** (billed per token beyond your quota)

Total calls are shown in the top-right corner. Instantly see usage spikes and quiet days.

---

### Top Models + Daily Token Usage — Find out where the budget goes

![Top Models and Daily Token Usage](docs/screenshots/models-tokens.png)

**Top 12 Models** — horizontal bar chart showing call count per model.  
In the example above: `claude-4.6-sonnet` dominated with 622 out of 708 total calls.

**Daily Token Usage** — stacked bar chart per day, broken down into 4 metrics:
| Color | Metric | What it means |
|---|---|---|
| 🟠 Orange | **CacheR** (Cache Read) | Tokens read from prompt cache — cheapest |
| 🔴 Red | **CacheW** (Cache Write) | Tokens written to prompt cache |
| 🔵 Blue | **Input** | Fresh input tokens sent to the model |
| 🟢 Green | **Output** | Tokens generated by the model — most expensive |

Switch the model dropdown (top-right of the chart) to inspect any specific model's token profile.

## ✨ Full Feature List

- 📊 **Usage Dashboard** — daily calls, spending trends, quota bars
- 💰 **Exact Cost Tracking** — per-request costs exactly as shown on cursor.com
- 🧾 **Detailed Record Table** — date, type, model, CacheR/CacheW/Input/Output tokens, cost per call
- 📈 **4 Chart Types** — daily calls, daily cost, daily token usage (4 metrics), model distribution
- 👤 **Multi-Account Support** — strict data isolation per account; auto-detects account switches
- 📤 **CSV Export** — one-click export, Excel-compatible (UTF-8 BOM)
- 🌐 **i18n** — English / 中文, follows browser language
- 🌙 **Dark / Light Mode** — follows system or manual override
- 🔒 **100% Local** — no server, no API key, no data collection

## 🛠 Build from Source *(for developers only)*

> **Regular users: use the one-click install links above. This section is not needed.**

```bash
# 1. Clone
git clone https://github.com/troublebaker/Cursor-Spending-Monitor.git
cd Cursor-Spending-Monitor/cursor监控插件/wxt-dev-wxt

# 2. Install dependencies
pnpm install

# 3. Build
pnpm build

# 4. Load in Chrome
# chrome://extensions → Enable "Developer mode" → "Load unpacked" → select .output/chrome-mv3/
```

### Dev Commands

```bash
pnpm dev          # Start dev server (hot reload)
pnpm build        # Production build → .output/chrome-mv3/
pnpm zip          # Package for Chrome Web Store → .output/*.zip
pnpm compile      # TypeScript type check
```
